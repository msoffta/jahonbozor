### 📚 Терминология проекта:
1. **Список (List)** — это заказ одного конкретного пользователя, который состоит из множества позиций (`order_item`).
2. **newrow** — паттерн UI в таблицах: пустая строка (обычно в конце или начале), при заполнении которой происходит создание новой записи (inline-создание).

#### 2. Флоу создания нового списка (клик по `+`)
* Сразу после нажатия на `+` открывается красивое модальное окно (Dialog/Modal).
* В модалке пользователь должен выбрать клиента (поиск/селект).
* **Создание нового клиента**: 
  * Первым пунктом в выпадающем списке модалки должна быть кнопка «Создать нового пользователя».
  * При клике на нее модалка закрывается, происходит редирект на страницу пользователей (Clients), и страница плавно скроллится до DataTable `newrow` для немедленного ввода данных нового клиента.
* **Выбор существующего клиента**:
  * Если клиент выбран, происходит переход на страницу создания/редактирования списка для этого клиента.

#### 3. Страница заказа (Списка)
* Используется отображение используя DataTable с включенным enableNewRow.
* Поля для заполнения внутри таблицы:
  * Товар (Combobox)
  * Количество (Number input с кнопками +/-)
  * Описание (Text input)
  * Тип оплаты (Select: Наличные, Карта и т.д.) (Select)
* Все эти данные сохраняются в рамках одного списка привязанного к выбранному ранее юзеру.

#### 4. Страница «Все списки»
* Открывается при клике на саму кнопку «Списки» в Bottom Bar.
* По умолчанию отображаются списки только за сегодняшний день.
* **Фильтры**:
  * Выбор периода (за прошедшие дни, date range picker).
  * Фильтр по конкретному пользователю (селект).
  * Текстовый поиск (по названию товара, описанию или номеру списка).
* Используется DataTable для отображение этих списком при нажатии на какой то из созданных списков он переходит туда (Страница списка *как при создании* ) где может посмотреть список или добавить туда что то или изменить
